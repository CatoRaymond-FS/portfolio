Overview

This project implements a simple and secure user authentication system. It allows users to sign up, log in, and maintain their session via JWT (JSON Web Tokens). Passwords are securely hashed using bcrypt, ensuring data security. The system provides a RESTful API with clear routes for handling authentication processes.

Key Features

  • Signup and Login: Users can securely sign up and log in with their credentials.
  • Password Hashing: Passwords are hashed with bcrypt before storage in the database, ensuring they are never stored as plaintext.
  • JWT Authentication: Authentication is managed using JSON Web Tokens (JWT), ensuring secure and stateless sessions.
  • API Documentation: The API is structured to support user authentication routes, including signup, login, and user profile retrieval.

Tech Stack

This project is built using the following technologies:

  • Backend: Node.js, Express.js
  • Database: MySQL for storing user credentials
  • Password Security: bcrypt for password hashing
  • Authentication: JWT for secure authentication and token management

Installation Instructions

Clone the Repository

To start, clone the repository to your local machine:

git clone https://github.com/CatoRaymond-FS/loginWithAuth.git

Install Dependencies

After cloning the repo, navigate to the project directory and install the necessary dependencies:

npm install

Set Up Environment Variables

Ensure you create a `.env` file at the root of your project with the following environment variables:


    DB_HOST=localhost
    DB_USER=root
    DB_PASSWORD=password
    DB_NAME=auth_system
    JWT_SECRET=your_jwt_secret

Start the Development Server

To run the server in development mode, use the following command:

npm start

The API will be available at http://localhost:5000 (or the specified port).

Usage

Once the server is up and running, you can use the following routes:

  • POST /signup: Registers a new user. Requires email and password.
  • POST /login: Authenticates a user. Requires email and password. Returns a JWT token.
  • GET /profile: Retrieves the logged-in user's profile information. Requires a valid JWT token in the Authorization header.

Security Considerations

All passwords are hashed with bcrypt before being stored in the database. This ensures that even if the database is compromised, user passwords cannot be retrieved in plaintext. Additionally, JWT tokens are used for session management, providing secure authentication and authorization.

Important: Always keep the JWT secret and database credentials secure by using environment variables and a secure .env file.
